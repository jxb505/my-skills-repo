# 安全违章报告工作流

使用此技能将上传的安全生产违章 Excel 导出数据转换为一份包含图表引用和管理建议的中文 Markdown 报告。

## 所需输入
- 一份包含安全违章记录的 Excel 工作簿 (`.xlsx` 或 `.xls`)。
- 典型字段包括：`状态`, `违章区域`, `检查机构`, `检查人员类别`, `违章时间`, `违章人员性质`, `总包单位`, `分包单位`, `违章情况`, `违章类别`, `违章级别`, `分值`, `数据来源`, `条款内容`, `连带记分信息`。
- 如果工作簿在数据表上方包含标题行，请自动检测实际的表头行，而不是默认使用第 1 行。

## 默认输出
- 一份中文 `.md` 报告。
- 包含描述性分析和嵌入的图表。
- 使用管理咨询的语气：客观、直接、数据驱动、并具有行动导向。
- 为输出文件清晰命名，例如 `安全生产违章情况深度分析与改进建议报告.md`。

## 智能分析工作流 (Agentic Workflow)
当你判断用户意图是生成安全违章报告时，必须严格遵循以下**三步走**流程：

### 第一步：提取数据摘要 (Data Extraction)
首先运行脚本的 `--export-stats` 参数，获取数据的统计摘要：
```bash
python scripts/generate_safety_report.py /path/to/input.xlsx --export-stats
```
（仔细阅读脚本输出的 JSON 摘要，包含总违章数、高频违章主题、违章重灾区等关键数据）

### 第二步：深度思考与动态创作 (Dynamic Thinking)
根据第一步获取的统计数据（特别是 TOP 违章主题和区域），结合《安全分析框架》和自身的领域知识，动态撰写**针对性**的“根因诊断”和“行动清单”。
将你的分析结果组织为如下格式的 JSON，并使用代码工具将其保存为本地临时文件（例如 `dynamic_analysis.json`）：
```json
{
  "root_causes": [
    "动态生成的根因 1（必须结合具体的 TOP 违章主题来写）",
    "动态生成的根因 2",
    "动态生成的根因 3"
  ],
  "actions": [
    {
      "measure": "动态整改措施 1",
      "detail": "具体的做法和要求",
      "owner": "责任主体，如：XX分厂",
      "metric": "衡量指标"
    },
    {
      "measure": "动态整改措施 2",
      "detail": "具体的做法和要求",
      "owner": "责任主体",
      "metric": "衡量指标"
    }
  ]
}
```

### 第三步：组装最终报告 (Report Assembly)
将包含动态分析的 JSON 文件路径作为 `--dynamic-file` 传递给生成脚本，生成最终 Markdown 报告：
```bash
python scripts/generate_safety_report.py /path/to/input.xlsx --output /path/to/output.md --title "安全生产违章情况深度分析与改进建议报告" --dynamic-file dynamic_analysis.json
```

该脚本将自动清洗数据、画图并注入你撰写的动态分析，最终生成一份包含结论、根因诊断和改进建议行动清单的完整 Markdown 报告。

## 报告结构
除非用户提供了其他模板，否则请遵循以下结构：

1. **总体概况与核心判断**
   - 阐述数据周期、有效记录总数、总扣分情况以及高风险违章占比。
   - 解释这些数据对安全管理意味着什么。

2. **高频违章与重点区域画像**
   - 识别 TOP 违章类别/主题和 TOP 区域/单位。
   - 突出重复发生、低门槛、高频率的问题。

3. **人员性质、协力单位与数据来源分析**
   - 对比正式员工、协力人员和外来人员。
   - 探讨协力管理、准入培训、现场监护和甲方责任是否存在薄弱环节。
   - 通过数据来源分布，判断监管方式主要是事后视频回看还是事中现场制止。

4. **管理体系短板与根因诊断**
   - 涵盖风险预控与安全交底、核心制度执行、高风险作业审批、协力管理、安全培训以及安全文化等方面。

5. **改进建议与行动清单**
   - 提供短期专项整治、中期体系优化和长期文化建设的建议。
   - 尽可能包含责任人、行动项、衡量指标和建议的完成时限。

6. **附录/口径说明**
   - 注明是否排除了已删除的行，以及源文件是否为不含连带的数据。

## 质量控制规则
- 不要仅仅罗列统计数据；必须解释其管理含义和风险隐患。
- 将 A类/高分/重复违章/协力集中/视频发现占比高 视为高优先级的风险信号。
- 如果图表无法正常渲染中文字符，请设置兼容中文的 matplotlib 字体（如果环境支持）。
- 如果用户上传了现有的 DOCX 作为样式示例，请将其作为内容参考，除非用户明确要求，否则不要将其视作权威数据。
- 遇到缺失的字段时，请使用现有字段继续分析，并简要说明数据的局限性。
- 确保所有图表标题和标签使用中文。

## 自定义选项
仅在必要时询问或推断以下内容：
- 报告标题；
- 数据周期（如果 Excel 中未体现）；
- 公司/工厂名称；
- 是否包含或剔除已删除的行；
- 若 Excel 中信息足够，是否包含连带记分责任分析；
- 是否使用提供的样式参考文件。
